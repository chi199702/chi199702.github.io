[技术] 解析 ELF 文件

Chi
2024-08-17 (Updated: 2024-08-18)

ELF 是 Executable and Linkable Format 的缩写，这种格式被用来存储 Linux 系统下的二进制可执行文件。拿 C 文件来说，用高级语言编写的源代码被编译成可执行的二进制目标文件（以下简称 target）的格式就是 ELF 格式。也就是说不管是一个简单的 hello world 程序还是拥有成千上万行代码的大型项目，被编译后得到都是一个 ELF 格式的二进制文件。运行 target 就是让操作系统的程序调度器把它从硬盘加载到内存中，然后把 target 组织成一个进程，让 CPU 指向代码段的第一条指令，后面就交给 CPU 和操作系统来维护进程的运行了。需要提到的是，对于只进行了编译但未进行链接而得到的中间目标文件和最终的可执行文件来说，其实他们的存储格式都是 ELF，只是内容上会存在稍许区别，若要细究则需进一步了解编译和链接的过程。

本文会对 ELF 的组织结构进行详细解析，在阐述完概念以后，再通过对编译链接完成得到的 target 进行实际的解析，以达到对 ELF 文件结构的更加直观的展现，加深理解。

ELF 组织结构

我与父亲不相见已二年余了，我最不能忘记的是他的背影。那年冬天，祖母死了，父亲的差使也交卸了，正是祸不单行的日子，我从北京到徐州，打算跟着父亲奔丧回家。到徐州见着父亲，看见满院狼藉的东西，又想起祖母，不禁簌簌地流下眼泪。父亲说，“事已如此，不必难过，好在天无绝人之路！”

回家变卖典质，父亲还了亏空；又借钱办了丧事。这些日子，家中光景很是惨淡，一半为了丧事，一半为了父亲赋闲。丧事完毕，父亲要到南京谋事，我也要回北京念书，我们便同行。

到南京时，有朋友约去游逛，勾留了一日；第二日上午便须渡江到浦口，下午上车北去。父亲因为事忙，本已说定不送我，叫旅馆里一个熟识的茶房陪我同去。他再三嘱咐茶房，甚是仔细。但他终于不放心，怕茶房不妥帖；颇踌躇了一会。其实我那年已二十岁，北京已来往过两三次，是没有甚么要紧的了。他踌躇了一会，终于决定还是自己送我去。我两三回劝他不必去；他只说，“不要紧，他们去不好！”
